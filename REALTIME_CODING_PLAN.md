# 实时体育教学应用 Coding Plan（单人版）

## 1. 目标
- 从离线视频评估升级为实时摄像头评估。
- 首版只做单人场景：稳定识别、实时反馈、自动计次、训练报告。
- 复用现有 `utils` 模型与特征处理逻辑，不改坏 `0_7` 脚本流程。

## 2. MVP 功能范围
- 实时摄像头采集（React 前端）。
- WebSocket 实时发送图像帧并返回评估结果。
- 支持动作：`pushup`、`squat`、`situp`（可扩展）。
- 实时输出：动作类型、阶段、质量分、错误提示、纠正建议、计次。
- 会话结束生成报告：平均分、最佳分、错误分布、达成率。

## 3. 架构设计
- 前端：React + Vite + TypeScript。
- 后端：FastAPI + WebSocket + 现有 PyTorch/YOLO 模型。
- 推理链路：摄像头帧 -> YOLO 关键点 -> 滑动窗口 -> 动作/阶段/质量模型。
- 教学层：错误提示映射 + 阶段状态机计次 + 分数平滑。

## 4. 关键技术点
- 滑窗长度：60 帧（与项目既有特征长度一致）。
- 推理间隔：每 3 帧推理一次，降低压力。
- 分数平滑：EMA（`alpha=0.3`）。
- 错误去抖：同一错误连续出现 6 帧才提示。
- 计次策略：基于阶段闭环（经过 bottom 后回到 finish/ready 计 1 次）。

## 5. API/WS 设计
- `GET /api/health`：模型和设备状态。
- `GET /api/actions`：动作列表。
- `POST /api/realtime/session/start`：创建会话。
- `POST /api/realtime/session/{id}/stop`：停止并返回报告。
- `GET /api/reports/{id}`：查询报告。
- `WS /ws/realtime/{id}`：帧输入与结果输出。

## 6. 前端页面设计
- 实时训练页：
  - 摄像头预览
  - 动作选择（自动/指定）
  - 开始/停止按钮
  - 实时评分、阶段、计次、错误与建议
  - 会话结束报告摘要

## 7. 实施步骤
1. 搭建 FastAPI 基础服务与 Session 管理。
2. 封装模型运行时（YOLO + Action/Phase/Quality）。
3. 实现实时引擎（滑窗、平滑、去抖、计次）。
4. 搭建 React 页面与 WebSocket 客户端。
5. 联调并验证端到端实时流程。
6. 补充部署说明与常见问题排查。

## 8. 验收标准
- 能在浏览器中打开摄像头并实时得到评估输出。
- 单人场景下端到端延迟在本地可接受（建议目标 < 300ms）。
- 能稳定计次并生成会话报告。
- 无 CUDA 时可自动回退 CPU，功能不中断。

## 9. 运行计划
- 第 1 周：后端链路与实时推理跑通。
- 第 2 周：前端交互、报告展示、性能优化与回归测试。
